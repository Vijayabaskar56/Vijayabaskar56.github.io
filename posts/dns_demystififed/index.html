Type of DNS Records Types

A Record (Address Record)

Function: Returns a 32-bit IPv4 address. It’s commonly used to map hostnames to IP addresses. Example: If you type “example.com” in your browser, the A record helps resolve it to an actual IP address like 192.0.2.11.

AAAA Record (IPv6 Address Record)

Function: Returns a 128-bit IPv6 address. Used to map hostnames to IPv6 addresses. Example: Similar to A records but for IPv6 addresses.

CNAME Record (Canonical Name)

Function: Creates an alias for one name to another. The DNS lookup continues by retrying with the new name. Example: If “www” is a CNAME for “webserver,” accessing “www.example.com” will lead to “webserver.example.com.”

NS Records (Nameservers)

Function: Specifies the authoritative nameservers for a domain. Example: These records point to servers where all DNS records for a domain are stored.

MX Records (Mail Exchanger)

Function: Points a domain to email servers using SMTP. Example: When someone sends an email to “user@example.com,” MX records direct it to the appropriate mail server.

TXT Record

Function: Used for various authentication methods and verifying domain ownership. Example: Often used for SPF (Sender Policy Framework) records to prevent email spoofing.

PTR Record (Pointer)

Function: Used for reverse DNS lookup. Associates an IP address with its domain name. Example: Helps verify the authenticity of an IP address.

SRV Record (Service Locator)

Function: Specifies the location of services (e.g., SIP, XMPP) in the network. Example: Used for generalized service location1.
